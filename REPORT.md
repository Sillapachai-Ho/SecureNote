### JS Engine
๋JavaScripts Engine คือ โปรแกรมคอมพิวเตอร์ที่ทำหน้าที่แปลงโค้ดภาษา JavaScript (Source Code) ให้เป็นภาษาเครื่อง (Machine Code) ที่ซีพียู (CPU) เข้าใจ เพื่อประมวลผลคำสั่งต่างๆ เช่น การทำให้หน้าเว็บมีการโต้ตอบ (Interactive) โดยทำงานอยู่เบื้องหลังในเว็บเบราว์เซอร์หรือสภาพแวดล้อมอย่าง Node.js 

### Runtime
JavaScripts Runtime คือ JS Engine ร่วมกับ Web APIs หรือ Library ต่างๆ ที่ช่วยให้ JavaScript โต้ตอบกับภายนอกได้ เช่น เบราว์เซอร์ หรือ Node.js ที่รัน JS บนเครื่องเซิร์ฟเวอร์
ส่วนประกอบสำคัญของ JavaScript Runtime:
- JavaScript Engine: ส่วนที่ประมวลผลโค้ด (เช่น V8 Engine ใน Chrome/Node.js)
- Web APIs / Library: API ที่จัดเตรียมไว้ให้ เช่น การจัดการ DOM ในเบราว์เซอร์ หรือ File System ใน Node.js
- Callback Queue & Event Loop: กลไกจัดการการทำงานแบบ Asynchronous เพื่อให้โปรแกรมไม่ค้าง

### DOM (Document Object Model)
DOM คือโครงสร้างข้อมูลที่เบราว์เซอร์สร้างขึ้นเพื่อจำลองหน้าเอกสาร HTML ให้อยู่ในรูปแบบของ "โครงสร้างต้นไม้ (Tree Structure)" หรือวัตถุ (Objects)
- หน้าที่หลัก: ทำหน้าที่เป็น "สะพานเชื่อม" ระหว่างโค้ด JavaScript กับหน้าจอเว็บไซต์
- การทำงาน: ทำให้ JavaScript สามารถเข้าถึงและปรับเปลี่ยนหน้าเว็บได้อย่างอิสระแบบเรียลไทม์ (Dynamic) เช่น การเข้าไปดึงข้อมูลจากช่อง Input, การเปลี่ยนสไตล์ CSS, หรือการซ่อน/แสดงองค์ประกอบบนหน้าจอตามการกระทำของผู้ใช้

### Environment Variables  
Environment Variables (ไฟล์ .env) คือตัวแปรพิเศษที่ใช้เก็บค่าคอนฟิกต่างๆ โดยจะถูกเก็บแยกไว้ "ภายนอก" ซอร์สโค้ดของแอปพลิเคชัน
- ด้านความปลอดภัย (Security): ใช้สำหรับจัดเก็บข้อมูลความลับขั้นสูง (Sensitive Data) เช่น Database Token, API Key, หรือรหัสผ่าน เพื่อป้องกันไม่ให้ข้อมูลเหล่านี้หลุดรอดไปอยู่ในระบบจัดการเวอร์ชัน (เช่น GitHub)
- ด้านความยืดหยุ่น (Configuration): ช่วยให้แอปพลิเคชันสามารถปรับเปลี่ยนพฤติกรรมตามสภาพแวดล้อมที่นำไปรันได้ทันที (เช่น สลับ URL ของ API ระหว่างเครื่อง Local กับเซิร์ฟเวอร์ Production) โดยที่ไม่ต้องเข้าไปแก้ไขซอร์สโค้ดด้านใน
- การเรียกใช้งาน: ฝั่ง Backend (Node.js) มักจะเรียกใช้ผ่าน process.env ในขณะที่ฝั่ง Frontend (เช่น Vite) จะเรียกใช้ผ่าน import.meta.env
